Azure Tutorial Part 3: Build the Pipeline

1. Create a new Pipeline

  1. Refresh the Repos page to confirm your code was successfully pushed to it.
  2. Click then .
  3. Click at the bottom of the page.
    • This gives you the option to create a pipeline through a GUI (as opposed to only through a YAML file).
  4. Keep selected and click .
  5. On the template selection screen, click at the top of the page.
  6. (Optional) A classic pipeline will add the -Cl suffix to the pipeline title by default. Change it to -Classic for clarity's sake.
  7. (Optional) Change the default job's name from Agent job 1 to something more descriptive (i.e., Build & Run).

2. Add Tasks: NuGet Restore & Visual Studio Build

NuGet Restore

  1. Click the on the job ().
  2. for and NuGet. By default, it should add as NuGet restore.
    • This pipeline step will install (a.k.a. “restore”) our project's dependancies on the pipeline agent so the code can properly run.
  3. Under Path to solution, packages.config, or project.json, browse () to in the root of your repo.

Visual Studio Build

  1. Click the on the job ().
  2. for and Visual Studio build.
  3. Under Solution, browse () to in the root of your repo.

3. Add Task: Command Line

  1. Click the on the job ().
  2. for and Command line.
  3. (Optional) Choose a more descriptive Display name (i.e., Output » txt).
  4. Under Script, enter .\path\to\MainApp.exe » path\to\TestCases.dll's\directory.
    • Example: .\MainApp\bin\Debug\netcoreapp3.0\MainApp.exe » TestCases\bin\Debug\netcoreapp3.0
    • Note: The .\ start to MainApp.exe's filepath is needed to make MainApp.exe execute.

4. Add Task: Visual Studio Test

  1. Click the on the job ().
  2. for and Visual Studio Test.
  3. (Optional) Choose a more descriptive Display name (i.e., Test CharN).
  4. Under Test files, enter path\to\TestCases.dll.
    • Example: TestCases\bin\Debug\netcoreapp3.0\TestCases.dll

6. Change a Test to Fail & Re-run the Pipeline

Change a Test to Fail

  1. Open your TestCases project in Visual Studio.
  2. In the Char5 [TestMethod], change the <expected letter> from “o” to something else (i.e., “l”):
    [TestMethod]
  public void Char5()
    {
     letter = txtOutput.Substring(4, 1);
     Assert.AreEqual(letter, "l");
    }
  3. In a terminal within the repo's root, run git add ., git commit -m “your_comment_here”, and git push.

Re-run the Pipeline & View the Results

  1. Click and click your pipeline under Recently run pipelines.
  2. Click .

  4. On the results page, click Tests underneath the run's title.
  5. Passed tests don't appear by default. To enable them, click > .

7. Configure Continuous Integration (CI)

Now let's enable Continuous Integration (CI). CI will make the pipeline automatically run whenever code is pushed to the specified branch on the remote repo.

  1. Click .
  2. Hover over your pipeline to reveal the More… kebab menu ().
  3. Click > .
  4. Click the Triggers tab > Enable continuous integration.
  5. Click the Tasks tab > Pipeline.
  6. Under Agent Specification, choose windows-2019.
    • This makes windows-2019 the default agent whenever the pipeline is triggered. The latest Windows agent is necessary for our app to run.
  7. Click > .
  8. Add a descriptive Comment and click .
