sawtooth:walkthroughs:jenkins_aws

As a Broad overview, creating a CI/CD pipeline with Github, Jenkins, and AWS requires the following:

  • Unordered List ItemGetting at least a proof-of-concept .Net Core application into a git repository on Github as a remote repository
  • Getting a Jenkins server set up for executing the build and publishing stages of the .Net Core application
  • Creating Infrastructure in AWS for the application to run on once deployed
  1. Getting application source into Github
  While individual developers will be committing code changes to the application, it will likely be helpful for the repository to be owned by a "Service account" - one that has its own credentials and can be managed on the organization level rather than being tied to an individual.
  1. Create a new generic github account
  2. Create a new repository using the new github account - make sure the repository is set to “private” for security purposes
  3. Add a personal github account as a contributor to the repository
  4. Log into the personal github account and clone the new repository onto a workstation where the proof-of-concept .Net Core application is stored
  5. Duplicate the proof-of-concept project's source directory into the new repository's location
  6. Add & commit the source code that has been added to the repository and push it to the remote server (on github)
  1. Configure a Jenkins Server
  Since the infrastructure for deployment will be in the AWS cloud environment, it makes sense that the Jenkins server ought to be hosted in this cloud service as well. Since this Jenkins server will be building a .Net Core application and Microsoft's build tools work best on Windows, we will be using Windows as the base operating system on the server where Jenkins will be installed.

Set up the server:

  1. In AWS, create a new EC2 instance in the desired region - make sure the server has at least 4GB RAM and is running a current version of Windows Server
  2. AWS protects administrative passwords to Windows servers with key pairs. Generate a key pair in the AWS Console for authenticating with the server.
  3. Create an AWS IAM user that has full S3 permissions - configure it as a service account“ with access and secret keys instead of passwords.
  4. Once you have credentials for the AWS server, open the Remote Desktop Connection application on a Windows computer. Use the “Public DNS” from the AWS Console to connect. Enter the administrator password from the AWS console you were able to retrieve with your key pair.
  5. Once connected, install the latest Microsoft .Net SDK, Microsoft Build Tools, Git for Windows, and Jenkins from their respective websites.
  6. Follow default installation procedures for Jenkins - once installed and started, the web interface should be accessible locally from the server itself at localhost:8080.
  7. In order to access the Jenkins web interface from a remote location (a browser of a personal computer, for example), create an AWS Security Group that has a rule to allow traffic from anywhere (0.0.0.0/0) to port 8080. Assign the Jenkins server EC2 instance to this security group and make sure it does not belong to other security groups that block port 8080.
  8. Log into the Jenkins web interface (Use the EC2 Public DNS address, appending ”:8080“. Log in with the default Jenkins credentials.
  9. Immediately change the default credentials to a new account and secure password.
  10. Once logged in, select 'Manage Jenkins', then 'Manage Plugins'
    1. Add the 'S3 Publisher' plugin. Accept defaults to install it.
  11. Go back to 'Manage Jenkins,' and select 'Configure System'
    1. Add the public DNS address from the EC2 instance to the 'Jenkins URL' field
    2. Add the name, access key, and secret key for the AWS IAM account you created in AWS for giving access to S3 services
  12. Go back to 'Manage Jenkins,' and select 'Global Tool Configuration'
    1. Add the direct path on the Windows Server for the “git” executable (i.e. C:\Program Files\git\bin\git.exe). This allows commands in Jenkins that use Git to know where the executable
  13. Go to 'Credentials' on the sidebar
    1. Add the “Service Account” github credentials

Build the pipeline:

  1. Click 'New Item' in the left sidebar
  2. Select 'Freestyle Project' and name it something descriptive (“build project”)
  3. Once the Project is created, select 'Configure' from the left sidebar (while in the Project view)
    1. Under 'Source Code Management', select 'Git' and add the URL for the github repository. Select the github credentials you added to Jenkins 'Credentials'
    2. Under “Branches to build” enter '*/master' to build from the master branch
    3. Under 'Build' we will enter a series of Windows batch commands. For each, press 'Add build step'>'Execute Windows batch command'
      1. In the first step, the project needs to be restored. Enter 'dotnet restore' in the Command window.
      2. In the second step, the project needs to be built. Enter 'dotnet build' in the Command window.
      3. In the third step, the build needs to be published with the 'Release' configuration as a self-contained Linux application. Enter 'dotnet publish –configuration Release –output C:\Users\Administrator\Desktop\LatestArtifact –self-contained -r win-x64' into the Command window.
      4. In the fourth window, the published files need to be zipped (compressed) to an easily-accessible location. Enter '“C:\Program Files\7-Zip\7z.exe” a ReadytoUpload\LatestArtifact.zip C:\Users\Administrator\Desktop\LatestArtifact\*' in the Command Window.
    4. At this point, every time the build pipeline is run, the application will be built, published, zipped, and saved to the Desktop of the Jenkins EC2 Instance.
    5. Under 'Post-build Actions', we will only add one step - copying the zipped folder to an S3 bucket in AWS.
      1. In the AWS Console, create a new S3 bucket and title it something unique and meaningful.
      2. Back in the Jenkins Web Interface, click 'Add post-build action' and select 'Publish artifacts to S3 bucket'.
        1. Select the S3 Profile you created when configuring Jenkins earlier
        2. Enter 'ReadytoUpload\*' into the 'Source' field (this is the folder we saved the zip file to earlier)
        3. Enter the name of the S3 bucket you created in the 'Destination Bucket' field
        4. Select the correct 'Bucket Region'
    6. Click 'Save' to save the Jenkins pipeline “project”
