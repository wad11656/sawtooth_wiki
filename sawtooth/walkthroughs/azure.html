Azure Tutorial Part 1: Create the Repo

1. Create an Azure DevOps Account

  1. Navigate to dev.azure.com.
  2. Click
  3. Choose an existing Microsoft account OR click No account? Create one!
  4. (If creating new MS account) Enter your email, password, e-mailed verification #, and CAPTCHA
  5. Select the United States region

2. Rename the Organization and Create a Project

  1. Click in the bottom left to bring up the organization Overview.
  2. Enter a new Name.
  3. Choose the correct Time Zone and click Save.
  4. Re-enter your new organization name to confirm changes.
  5. Enter your Project name (Hint: Avoid using special characters) and click .

3. Create an SSH Key Pair (Public Key + Private Key)

  1. Click your profile icon in the top right () and click the ellipses More options icon ().
  2. Click then . Stay on this page for Step 4.
  3. Generate the private+public key pair:
    1. Open a terminal (MacOS/Linux) or Git BASH (Windows)
    2. Run ssh-keygen -t rsa
    3. Enter the filepath you want the key saved in.
      • Default filepath = \Users\your_username\.ssh\id_rsa, where id_rsa and id_rsa.pub will be the private and public key filenames.
    4. (Optional) Secure you SSH key pair with a password when prompted.

If you secured your SSH key pair with a password & don't want to enter it every time you SSH:

  1. Start an ssh-agent to use on your machine, with eval ssh-agent.
  2. ssh-add your key, type your password once.
  3. Profit! You can now ssh to any host that has your public key without having to enter a password.

4. Copy your Public Key Data into Azure DevOps

  1. Navigate to the directory housing your key pair (Chosen in Step 3.3c).
  2. Open the the public key (Default = id_rsa.pub) in a text editor and copy its contents.
  3. Navigate to the Azure DevOps web page you opened in Step 3.2 and click .
  4. Name the key pairing associated with your computer. (For example, your computer's hostname.)
  5. Paste the contents you copied in Step 4.2 into the Public Key Data field.
  6. Click .

5. Clone the Azure DevOps Repo to your Computer

  1. Go the Azure DevOps landing page by clicking in the top left.
  2. Click your project tile on the landing page.
  3. Click in the left-hand navigation bar.
  4. Click in the upper-right of the Repos page.
  5. Copy the SSH address by clicking then .
  6. In a terminal (MacOS/Linux) or Git BASH (Windows), cd to the directory where you'd like to store the Azure DevOps repo on your computer.
    • Note: The clone command will automatically create a new folder with your repo's files in it, so no need to manually create that folder beforehand.
  7. Type git clone followed by a space, then paste and run the SSH address to clone the Azure DevOps repo onto your computer.
    • Example: git clone git@ssh.dev.azure.com:v3/humanbeing/SawtoothCapstone2019/demo_app
  8. cd into your newly-cloned local repo directory.
  9. Run ls -a in your terminal/Git BASH to show all files in the current directory (including hidden ones) and confirm you see the hidden .git directory.

6. Make & Push your First Commit

  1. Create (touch) a README.md file in your local repo's directory.
  2. Open the file with a text editor (i.e., nano README.md) and add a short description to the file.
  3. Save the file (ctrl+O in nano) and close the file (ctrl+X in nano).
  4. Run ls -a in your terminal/Git BASH to show all files in the current directory (including hidden ones) and confirm you see the .git directory and your newly-created README.md.
  5. Run git add . to add all files in the current directory–and in its subdirectories–to the Git Staging area.
  6. Run git commit -m “your_commit_comment_here” to wrap up all the Staged files into a single revision (a.k.a. a commit).
  7. Run git push origin master to push your committed files to the origin repo's master branch.
    • Note: The origin repo is the repo in Azure DevOps, which you originally cloned from.
  8. Refresh your repo's web page in Azure DevOps and check if your newly-pushed README.md is listed in the branch.

7. (Optional) Omit Unwanted System Files from Git Commits

If you have unwanted system files showing up in your repos…

  1. Delete all unwanted files from your local repo's directory and subdirectories.
  2. Create a new file called .gitignore_global. (A good place is in the root of your user's folder.)
  3. List all the names of files you'd like to omit from all future git commits.
  4. Run git config –global core.excludesfile /path/to/.gitignore_global to set your file as the global gitignore file for your git installation.
  5. Push your local repo to the Azure DevOps master branch by following Steps 6.5 - 6.8.
  6. Test the functionality of your .gitignore_global by refreshing your Azure DevOps repo web page and ensuring the unwanted system files are no longer in the current master branch revision.

  2. Click .

  4. Select your repo from the dropdown and click .
