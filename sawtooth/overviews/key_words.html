Key Terminology

Cloud-Native Application

A collective of online computation services that, together, deliver a defined result.

  • Cloud-native applications are commonly comprised of microservices, which are online services that can each be upgraded, scaled, and deployed independent from one another.
  • Containers are a common implementation, which use OS-level virtualization to create app-hosting platforms that are far more light-weight than virtual machines (VM).
  • Cloud-native applications are closely tied to the DevOps model, and therefore the Continous Integration and Continuous Delivery (CI/CD) of software, meaning regular and consistent software testing and delivery through automation.

Continuous Integration / Continuous Deployement (CI/CD) Pipeline

An automation-centered development workflow focused on regular code integration and regular deployment to the end user.

  • Continuous Integration (CI) is the practice of integrating and testing developers' code changes regularly (sometimes multiple times a day). This way, bugs and code breaks are found much faster and, theoretically, with a much more manageable amount of time to resolve them.
  • Continuous Deployment/Delivery (CD) is the practice of deploying/delivering new builds of the product to the end user regularly and as soon as they are ready. Delivering software updates to end users on a regular schedule makes updates less momentous or risky and leads to the ability to fix user-reported bugs quickly (more specifically, simply on the next update).

Cloud Services

Online software, storage, and compute services used as an alternative to on-premises solutions for the same functions.

  • Cloud Services are run on the infrastructure of some cloud provider (i.e., Amazon Web Services (AWS), Google Cloud) as opposed to on on-premises hardware.
  • Cloud services are closely associated with the various services cloud providers offer, the primary ones being: Software as a Services (SaaS), Platform as a Service (PaaS), and Infrastructure as a Service (IaaS). Each of these types of services offer varying levels of abstraction to the development team that uses them. The more abstracted the service, the less upkeep the development team needs to worry about.
    • SaaS provides software that's already deployed and functioning online, such as Dropbox or Gmail. Target: Standard Consumers
    • PaaS, as the name implies, is a platform for developers to host and run their code online, therefore creating their own SaaS for others to use. There are minimal to no OS or hardware customization options, as almost all of that is managed by the cloud service provider. Target: App Developers
    • IaaS offers full OS and hardware customization to host your cloud services on. Target: Infrastructure & Network Architects (& App Developers)

Infrastructure as Code

Infrastructure as Code, or IaC, is the use of descriptive files to manage and deploy infrastructure (networking, load balancers, VMs, etc.), as opposed to physically configuring the infrastructure or using configuration software.
