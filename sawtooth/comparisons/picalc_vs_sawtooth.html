PiCalc vs. Sawtooth Apps

In order to test and document pipelines, we, the BYU capstone team, originally committed to creating “dummy”/demo apps that weren't necessarily representative of Sawtooth Software's current applications. However, in the second semester of development, the capstone team was asked by the professor to go a step further and–if we were unable to use Sawtooth's actual programs in our pipelines–to at least produce an application as closely akin to Sawtooth's in-house software as possible.

Therefore, we designed a .NET application called “PiCalc” and integrated it with a pipeline. PiCalc is a calculator for the number pi, and can take 2 parameters:

  1. The number of decimal points to calculate
  2. The number of times to output the result

Similar to the applications Sawtooth uses in-house, PiCalc:

  • …is written in C# with standard C# libraries.
  • …is entirely backend (no UI).
  • …is scalable to be computationally intense (takes processing power).
  • …outputs numeric values in JSON format.
  • …is stateless (just gets input and spits out output).
